# セキュリティポリシー (SECURITY.md) - common_lib

[English](../en/SECURITY.md) | **日本語版**

`common_lib` のセキュリティ方針、サポート対象バージョン、および構成要素に関するセキュリティ上の考慮事項について説明します。

---

## 1. サポート対象バージョン

現在、セキュリティアップデートは最新のリリースブランチに対してのみ提供されます。

| バージョン | サポート状況 |
| :--- | :--- |
| >= 0.1.x | サポート対象 |
| < 0.1.0 | 非サポート |

---

## 2. 脆弱性の報告手順

`common_lib` にセキュリティ上の脆弱性を発見された場合は、パブリックな Issue を作成せず、非公開にてプロジェクトメンテナーまで報告してください。

- **対応時間**: 報告を受け取ってから 48 時間以内に受領確認を行い、対応計画および修正方針を回答します。

---

## 3. セキュリティ上の考慮事項

### Named Mutex の利用 (Windows 二重起動防止機能)

二重起動チェック機能（`check_single_instance` および `desktop::acquire_single_instance`）は、Windows の Named Mutex を利用しています。本機能を利用する開発者は以下の点にご注意ください：

1. **Mutex の乗っ取り (サービス拒否攻撃 / DoS)**:
   - Named Mutex は共有ネームスペース上に存在します。同一マシン上の別プロセスが予測可能な名前であらかじめ Mutex を生成した場合、本アプリの起動が阻害される可能性があります。
   - **推奨対策**: 予測しやすい汎用的な Mutex 名を避け、企業ドメインの逆順表記（例: `"com.mycompany.myapp.mutex_name"`）や GUID などをプレフィックスに含めることを推奨します。
2. **アクセス制御 (Security Descriptor)**:
   - 本ライブラリの `CreateMutexW` はデフォルトのセキュリティ記述子（`None`）で初期化されています。マルチユーザー環境（ターミナルサービスやリモートデスクトップ等）では、デフォルトでユーザーセッションローカルとなります。
   - 全ユーザー間でシステムワイドな単一インスタンスを保証したい場合は、Mutex 名の先頭に `"Global\\"` を付与する必要がありますが、適切な権限設定が必要となります。プレフィックスなしの場合はセッションローカル（`"Local\\"`）として動作します。
