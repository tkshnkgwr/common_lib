# Project Roadmap & Todo Management (TODO.md)

本プロジェクト `common_lib` の機能実装状況、直近で対応すべきタスク、および将来的な拡張機能のアイデア（バックログ）を管理します。

---

## 1. 実装済みの機能 (Done)

### コア機能
- **二重起動防止機能 (Named Mutex チェック)**:
  - Windows API (`CreateMutexW`) による多重起動防止の仕組み (`check_single_instance`)。
- **二重起動防止機能 (RAII ガード方式)**:
  - アプリ生存期間を通じてロックを安全に維持し、終了時に自動解放する `SingleInstanceGuard` および `acquire_single_instance`。
  - 非Windows環境におけるダミー構造体およびダミースタブの提供。
- **テキスト差分計算**:
  - 動的計画法（LCS: 最長共通部分列）を利用した、行単位のテキスト比較・差分検出エンジン (`compute_diff`)。
- **文字列ユーティリティ**:
  - 大文字小文字を区別しない単語出現回数カウント (`count_occurrences`)。
  - バイト数データを人間が読みやすい形式 (`B`, `K`, `M`, `G`) に変換する機能 (`format_bytes`)。
  - タイトル、本文、説明の重要度スコア（タイトル出現は重み2倍）を計算するタグ提案アルゴリズム (`suggest_tags`)。

### 開発・品質管理基盤
- **テスト自動化**:
  - 各APIに対応するユニットテストの整備。
  - Rustdocのコメント例をコンパイルおよびテスト検証するドキュメンテーションテスト（Doc-tests）の整備。
- **CI/CD環境**:
  - GitHub Actions によるビルド・テスト・Clippy・Rustfmtの自動チェックワークフローの構築。
  - リリースバイナリの自動生成・Releaseページの自動作成機能。
- **ドキュメンテーション**:
  - 英語・日本語の README、機能仕様書 (`SPEC.md`)、構成図 (`DIAGRAM.md`)、利用例集 (`EXAMPLES.md`)、パフォーマンス分析 (`FOOTPRINTS.md`)、セキュリティ方針 (`SECURITY.md`)、貢献者ガイド (`CONTRIBUTING.md`) の作成。

## 3. 将来の拡張提案 (Backlog)

- **他OS環境における二重起動防止機能の実装**:
  - 現状ダミー実装になっている Linux / macOS 環境向けに、ファイルロック（`flock`）や UNIX ドメインソケットを用いた、安全なクロスプラットフォーム二重起動防止の仕組みの実装。
- **テストカバレッジの可視化**:
  - Codecov などのツールを CI ワークフローに組み込み、テストカバレッジを自動計測してバッジを表示する機能。
- **簡易形態素解析によるタグ自動提案の高度化**:
  - 現在は完全一致での単語カウントのみを行っている `suggest_tags` について、より日本語などの自然言語に配慮した簡易的な単語境界分割処理や、TF-IDF のような重み付けロジックの導入。
